# Linux Specific SPB Notes

## Using bubblewrap to somewhat isolate the file system

  - Only somewhat isolating the file system
  - To access to spb templates, specifically bind the ```~/bin/spb-tempatle``` directory
      - Alterativly, bind a specific SPB template to prevent access to all SPB template data
  - Depending on your particualr distribution and installation meathod for Brave you may need to make modifications
  - Depending on usage bwrap can use significant ammounts of memory. Keep an eye on your usage.

```
# create fake group file
cat > /tmp/fake-group << EOF
group:x:$(id -g):
EOF

# create fake password file
cat > /tmp/fake-passwd << EOF
user:x:$(id -u):$(id -g)::/home/user:/bin/bash
EOF

# run bubble wrap - isolation (file system only) - ready for use with brave browser
bwrap \
--ro-bind /tmp/fake-passwd /etc/passwd \
--ro-bind /tmp/fake-group /etc/group \
--ro-bind $HOME/bin/start-private-browser.bash /home/user/bin/start-private-browser.bash \
--ro-bind /etc/os-release /etc/os-release --ro-bind /usr /usr \
--ro-bind /bin /bin \
--ro-bind /lib /lib \
--ro-bind /etc/alternatives /etc/alternatives \
--ro-bind /opt/brave.com /opt/brave.com \
--ro-bind /lib64 /lib64 \
--ro-bind /etc/resolv.conf /etc/resolv.conf \
--ro-bind /etc/ssl /etc/ssl \
--ro-bind /etc/fonts /etc/fonts \
--ro-bind /etc/localtime /etc/localtime \
--dev-bind /dev /dev \
--proc /proc \
--tmpfs /tmp \
--tmpfs /run \
--dir /tmp/spb-brave-file-system \
--setenv HOME /home/user \
--setenv XDG_RUNTIME_DIR /run \
--share-net \
--unshare-pid \
--die-with-parent \
bash

# within your bubble wrapped bash start SPB
/home/user/bin/start-private-browser.bash
```
